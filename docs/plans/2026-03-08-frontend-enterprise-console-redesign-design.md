# Frontend Enterprise Console Redesign Design

**Date:** 2026-03-08

**Goal**

将当前前端从“模板化 AI 工作台”重构为更接近企业运维控制台的体验，先用 Figma 效果稿确认视觉和信息架构，再进入正式实现。

## Background

当前界面存在几个主要问题：

- 首页更像产品宣传页，不像运维总控台。
- 故障分析页信息密度高但阅读路径不清晰，Tabs 和对话流互相抢主视觉。
- 卡片、渐变、标签和留白的组合偏通用 SaaS 模板，缺少 SRE 场景下的专业感。
- 工具调用、责任田映射、主 Agent 命令、专家反馈和最终结论没有形成明确的层级。

## Design Direction

采用“企业控制台”方向，而不是战情大屏或营销式首页。

设计目标：

- 克制、专业、稳定，去掉明显的“AI 生成味”。
- 先结论，再证据，再原始细节。
- 页面分区按工作流组织，不按功能按钮堆叠。
- 保留 Ant Design 组件体系，重构布局、层级、密度和视觉 token。

## Information Architecture

### 1. 首页

首页改为总控台：

- 顶部：关键指标条
- 左侧主区：快速创建分析
- 右侧侧区：系统状态与接入状态
- 下半区：进行中事件、最近完成分析
- 底部：Agent 能力矩阵

### 2. 故障分析页

故障分析页改为调查工作台：

- 左栏：事件输入、会话控制、运行状态
- 中栏：辩论过程主工作区
- 右栏：责任田映射、关键结论、风险与建议
- 底部折叠区：原始事件明细与技术日志

### 3. 责任田页

- 资产总览
- 导入与维护

### 4. 历史页

- 固定筛选条
- 单一滚动区域
- 默认突出根因摘要、状态和更新时间

### 5. 设置 / 高级页

- 分组成“模型与运行时 / 数据源与工具 / 评测与治理 / 安全与审计”

## Dialogue Design

对话流分四类对象：

- 主 Agent：命令与阶段判断
- 专家 Agent：分析结论与证据
- 工具调用：嵌入式证据条，不作为普通聊天气泡
- Judge / 报告：阶段性收敛卡片

每条消息默认只显示：

- Agent 名称、时间、状态
- 一句摘要
- 可展开的完整分析与证据

## Visual System

- 主背景：冷白偏灰
- 主色：深蓝灰
- 强调色：低饱和青蓝
- 风险色：锈红、琥珀
- 卡片圆角收紧到 10-12px
- 降低渐变和悬浮阴影占比

## Mockup Scope

本轮先输出两张高优先级效果稿：

1. 首页总控台
2. 故障分析工作台

实现方式：

- 在 `frontend/public/figma-mockups/` 下生成静态 HTML/CSS mock 页面
- 用 Figma MCP 抓取本地页面生成设计稿
- 设计确认后再进入 React 页面重构
